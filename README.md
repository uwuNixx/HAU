# labHAU - Учёт заявок в ЖКХ

Spring Boot приложение для управления заявками жильцов в системе ЖКХ.

## Запуск

### 1. Профиль `dev` (H2 in-memory - БД не требуется)

```bash
mvn spring-boot:run
```

Swagger UI: http://localhost:8080/swagger-ui.html

### 2. Профиль `pg` (локальный PostgreSQL)

PostgreSQL должен быть запущен на `localhost:5432` и база `labhau_db` должна быть создана.

```bash
mvn spring-boot:run -Dspring-boot.run.profiles=pg
```

## Технологии

| Компонент | Технология |
|-----------|-----------|
| **Язык** | Java 17 |
| **Фреймворк** | Spring Boot 3.4.1 |
| **База данных** | PostgreSQL (основная), H2 (для тестов/dev) |
| **ORM** | Spring Data JPA / Hibernate |
| **Миграции** | Liquibase |
| **Безопасность** | Spring Security + JWT (jjwt 0.12.6) |
| **Маппинг DTO** | MapStruct 1.6.3 |
| **Валидация** | Jakarta Validation |
| **Документация API** | SpringDoc OpenAPI (Swagger UI) |
| **Сборка** | Maven |
| **Файловое хранилище** | Файловая система сервера |

## Роли пользователей

- **RESIDENT** — жилец (может создавать заявки, смотреть свои)
- **DISPATCHER** — диспетчер (просмотр всех заявок, изменение статуса, файловые операции)
- **ADMIN** — администратор (всё что может диспетчер + удаление сущностей, статистика)

## Основные эндпоинты

### Аутентификация (`/api/v1/auth`)
| Метод | Путь | Доступ | Описание |
|-------|------|--------|----------|
| POST | `/login` | Публичный | Вход по email+password, получение JWT |
| POST | `/register` | Публичный | Регистрация нового жильца (ФИО + адрес дома/квартиры) |

### Заявки (`/api/v1/applications`)
| Метод | Путь | Доступ | Описание |
|-------|------|--------|----------|
| GET | `/` | ADMIN, DISPATCHER | Список всех заявок с фильтрацией |
| GET | `/my` | RESIDENT | Заявки текущего жильца |
| GET | `/stats` | ADMIN | Статистика по статусам |
| GET | `/{id}` | Авторизованные | Детали заявки |
| POST | `/` | RESIDENT | Создать новую заявку |
| PATCH | `/{id}/status` | ADMIN, DISPATCHER | Изменить статус заявки |
| DELETE | `/{id}` | ADMIN | Удалить заявку |

### Дома (`/api/v1/buildings`)
| Метод | Путь | Доступ | Описание |
|-------|------|--------|----------|
| GET | `/` | Авторизованные | Список домов |
| GET | `/{id}` | Авторизованные | Детали дома |
| GET | `/{id}/applications` | ADMIN, DISPATCHER | Заявки по дому |
| DELETE | `/{id}` | ADMIN | Удалить дом |

### Квартиры (`/api/v1/apartments`)
| Метод | Путь | Доступ | Описание |
|-------|------|--------|----------|
| GET | `/{id}` | ADMIN, DISPATCHER | Детали квартиры |
| GET | `/my` | RESIDENT | Своя квартира |
| DELETE | `/{id}` | ADMIN | Удалить квартиру |

### Категории услуг (`/api/v1/service-categories`)
| Метод | Путь | Доступ | Описание |
|-------|------|--------|----------|
| GET | `/` | Авторизованные | Справочник категорий |
| GET | `/{id}` | Авторизованные | Детали категории |

### Пользователи (`/api/v1/users`)
| Метод | Путь | Доступ | Описание |
|-------|------|--------|----------|
| DELETE | `/{id}` | ADMIN | Удалить пользователя |

### Файлы (`/api/v1/files`)
| Метод | Путь | Доступ | Описание |
|-------|------|--------|----------|
| POST | `/upload` | ADMIN, DISPATCHER | Загрузить .json с данными заявки → создаётся заявка |
| GET | `/{applicationId}` | ADMIN, DISPATCHER | Скачать заявку в .json |
| DELETE | `/{id}` | ADMIN, DISPATCHER | Удалить файл из хранилища |

## Сущности (БД)

- **User** — пользователь (email, пароль, ФИО, телефон, роли)
- **Role** — роль (RESIDENT, DISPATCHER, ADMIN)
- **Building** — жилой дом (город, улица, номер, управляющая компания)
- **Apartment** — квартира (номер, этаж, привязка к дому и жильцу)
- **Application** — заявка (заголовок, описание, статус, приоритет, даты, автор, исполнитель, квартира, категория)
- **ServiceCategory** — категория услуги
- **FileEntity** — файл (имя, тип, размер, путь на диске, привязка к заявке и загрузившему)
